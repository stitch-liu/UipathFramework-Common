*项目的框架说明：
 Component目录：这个目录存储的为项目配套的组件内容，用于已经进行组装好的元件存放。
 Core目录：该目录存放的是项目重要的控件，如为项目提供的专属组件，通常指的是能用于整体项目流程里面的核心控件。
 Data目录：用于存放所有项目相关的输出输入文件。
 DB目录：用于存放与项目数据库相关的方法（注意：这里不存放数据库基本的访问方法）。
 Documentation目录：存放与相关相关的文档信息，用于日后进行项目流程检查时候使用。
 Exceptions_Screenshots目录：用于存放与项目相关的截图信息。
 Extention APP目录：针对某一领域扩充的方法工具。
 Framework目录：该目录为项目的框架控件目录。
 Lib目录：该目录为扩展控件目录，存放所有的工作控件。
 Systems目录：针对某系统的流程进行组装的扩充控件。
 Tests目录：用于测试使用的目录。
